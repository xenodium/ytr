* YouTube Radio (ytr)

An Emacs package to stream YouTube audio (powered by [[https://github.com/yt-dlp/yt-dlp][yt-dlp]] and [[https://mpv.io][mpv]]).

#+HTML: <img src="https://raw.githubusercontent.com/xenodium/ytr/main/screenshot.gif" width="70%" />

* Install / Setup

#+begin_src emacs-lisp :lexical no
  (use-package ytr
    :vc (:url "https://github.com/xenodium/ytr" :rev :newest)
    :commands (ytr))
#+end_src

Then =M-x ytr= to open the player. The first time, you'll be prompted for a channel or playlist URL.

** Doom Emacs

Add the package to ~packages.el~

#+begin_src elisp
(package! ytr
  :recipe (:host github :repo "xenodium/ytr"))
#+end_src

Configure ytr in ~config.el~ to work with the SPACE leader key and evil mode.

#+begin_src elisp
(use-package! ytr
  :commands ytr
  :hook (ytr-mode . evil-emacs-state)
  :init
  ;; Optional: a leader binding to open the player
  (map! :leader :desc "YouTube radio" "o y" #'ytr))
#+end_src


* Usage

- =M-x ytr= toggles player display.
- Press =+= to add a YouTube channel or playlist URL, for example:
  - A channel: =https://www.youtube.com/@TOKYOECHOSOUNDS=
  - A playlist or mix: =https://www.youtube.com/playlist?list=...=
- Press =/= to pick any track across all your channels, or =s= to pick a channel and play its first track.
- =SPC= toggles play/pause, =n= / =p= move between tracks, and =f= / =b= seek.

* Player key bindings

| Key       | Command             | Description                                          |
|-----------+---------------------+------------------------------------------------------|
| =SPC=       | ytr-toggle-play     | Toggle play/pause, or resume the last played track.  |
| =n=         | ytr-next            | Play the next track in the catalog.                  |
| =p=         | ytr-previous        | Play the previous track in the catalog.              |
| =f=         | ytr-seek-forward    | Seek forward (5 seconds by default).                 |
| =b=         | ytr-seek-backward   | Seek backward (5 seconds by default).                |
| =/=         | ytr-play-track      | Select a track from all channels and play it.        |
| =c=         | ytr-play-channel    | Select a channel and play its first track.           |
| =+=         | ytr-add-channel     | Prompt for a channel/playlist URL, add it, and play. |
| =-=         | ytr-remove-channel  | Select a channel and remove it from the catalog.     |
| =o=         | ytr-open-in-browser | Open the current track in a web browser.             |
| =TAB=       | ytr-forward-button  | Move point to the next button.                       |
| =<backtab>= | ytr-backward-button | Move point to the previous button.                   |
| =q=         | ytr-quit            | Hide the player (playback continues).                |
| =Q=         | ytr-exit            | Stop playback and close the player.                  |

With a numeric prefix, =f= / =b= seek that many seconds; with a plain prefix (=C-u=), they seek in minutes.

* External dependencies

=ytr= requires a graphical Emacs (it renders images in a child frame), plus two command line tools in your =exec-path=:

- [[https://github.com/yt-dlp/yt-dlp][yt-dlp]]: fetches channel/playlist listings.
- [[https://mpv.io][mpv]]: plays the audio.

On macOS:

#+begin_src sh
  brew install yt-dlp mpv
#+end_src

* Customizations

** Data directory

By default, ytr keeps its state file and cached thumbnails under =ytr/= in your =user-emacs-directory=.  Point =ytr-data-directory= elsewhere to keep things tidy:

#+begin_src emacs-lisp :lexical no
  (setq ytr-data-directory "~/.cache/ytr/")
#+end_src

** Buttons (macOS SF Symbols)

If you have [[https://lmno.lol/alvaro/emacs-insert-and-render-sf-symbols][SF symbol rendering in Emacs]] on macOS, enable them in the player's buttons with:

#+begin_src emacs-lisp :lexical no
  (ytr-macos-use-sf-symbols)
#+end_src

** Button icons

Each button icon is a variable you can set to any string, for example:

#+begin_src emacs-lisp :lexical no
  (setq ytr-play-icon "▶"
        ytr-stop-icon "■"
        ytr-next-icon "▷▷"
        ytr-previous-icon "◁◁")
#+end_src
